# 范式对比与决策指南

## 对比

| 维度 | ReAct | DAG Workflow | Plan and Execute | 纯脚本 | ARG 行动链路 |
| --- | --- | --- | --- | --- | --- |
| Agent 是否决定下一步 | 是 | 否 | 常常是 | 否 | 否 |
| Agent 是否看到全局 | 是 | 通常是 | 是 | 不适用 | 尽量否 |
| 验证方式 | 弱 | 取决于实现 | 常为自检 | 强 | 强 |
| 灵活性 | 高 | 中 | 高 | 低 | 中低 |
| 可重复执行 | 中 | 高 | 中 | 高 | 高 |
| 适合开放探索 | 高 | 低 | 中 | 低 | 低 |
| 适合结构化流水线 | 中 | 高 | 中 | 高 | 高 |

## 决策树

```text
任务需要 LLM 的语义判断吗？
  否 -> 优先用纯脚本
  是 ->
    产物是否结构化？
      否 -> 用 ReAct 或人工协作
      是 ->
        是否需要重复执行？
          否 -> Plan and Execute 可能够用
          是 ->
            每步是否能验证？
              否 -> ARG + 人工确认
              是 -> ARG 行动链路
```

## 推荐使用场景

- 公开数据采集后分类
- 用户反馈归类
- 文档批处理
- 日报/周报生成
- 多来源数据汇总
- 需要定时执行的 Agent 任务

## 不推荐使用场景

- “帮我研究一下某领域”
- “和我一起头脑风暴”
- “写一篇有感染力的文章”
- “自由探索代码库并提出架构建议”

这些任务不是不能拆 step，而是硬拆后会损失主要价值。
