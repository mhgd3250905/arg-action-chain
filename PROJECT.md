# 项目定位

## 名称

ARG 行动链路

英文可写作：ARG Action Chain。

## 一句话

让 Agent 像 ARG 玩家一样，只读取当前线索卡，完成当前任务，通过验证后解锁下一步。

## 定义

ARG 行动链路是一种面向 LLM Agent 的受控执行范式。它将复杂任务拆解为一组只读 Step Contract，通过逐步揭示、契约执行、格子化语义判断和可执行验证门禁，降低 Agent 在多步任务中的超前规划、行动发散和自我验收风险。

## 不是什么

- 不是通用 Agent 框架。
- 不是提示词技巧合集。
- 不是让 Agent 更自由的规划系统。
- 不是替代脚本、队列、DAG、CI/CD 的万能工具。

它更像是一种工程纪律：承认 Agent 不稳定，然后把它放进有边界的执行协议里。

## 核心口号

- Agent 是玩家，不是出题人。
- Step Contract 是最小行动边界。
- 验证命令是唯一验收权威。
- 语义能力只在有限字段中发挥。

## 经验判断

ARG 行动链路不能消灭模型错误，但能明显限制三类问题：

- 发散：Agent 不再轻易跳出当前任务边界。
- 幻觉：Agent 的自由文本空间被压缩为有限字段、枚举和结构化输出。
- 欺骗：Agent 不能只靠自我声明完成任务，必须通过可执行验证。

这套范式的价值不是“让 Agent 更聪明”，而是“让 Agent 更难乱来”。
